# Marketing photo drop folder (internal)

Drop JPG/WebP files into `public/images/` using these exact names. They appear on the live site automatically — no code change required.

| Filename | Used on |
|----------|---------|
| `hero-poster.jpg` | Homepage hero background + social link preview (`og:image`, 1200×630 recommended) |
| `hero-video.mp4` | Homepage hero loop (optional; poster used as fallback) |
| `physician.jpg` | Homepage clinical leadership |
| `clinic-exterior.jpg` | Homepage gallery |
| `clinic-interior.jpg` | Homepage gallery |
| `iv-lounge.jpg` | Homepage gallery + IV banner |
| `service-hormones.jpg` | Homepage service card |
| `service-weightloss.jpg` | Homepage service card |
| `service-iv.jpg` | Homepage service card |
| `service-peptides.jpg` | Homepage service card |
| `editorial-hormones-women.jpg` | Women's hormones storefront |
| `editorial-hormones-men.jpg` | Men's health storefront |
| `editorial-peptides.jpg` | Peptides storefront |

Until files exist, **image slots are hidden on the live site** (no placeholder panels, no “add photo” UI). Text-only layouts remain. Sections such as the homepage clinic gallery do not render until at least one gallery file is present.

Recommended: landscape 16:9 for hero; 4:5 portrait for editorial columns; 16:10 for service cards.
